# フロントエンド ディレクトリ構成

## 設計方針

- `frontend/` 直下にページを 1 階層で並べる（深い入れ子を持たない）
- 各ページは `<ページ名>/index.html` + `style.css` + `script.js` の 3 点セット
- 認証系 (`auth/`) やマイページ系 (`mypage/`) などのグループディレクトリは持たない

## 変更のポイント

### `auth/` グループを解体
`login/`, `register/`, `verify/`, `recovery/`, `password-reset/` を直下に昇格。  
URL が `/auth/login` → `/login` と短くなり、リンク管理が簡単になる。

### `auth/recovery/reset/` → `password-reset/`
3 階層だったページを 1 階層に。`recovery/` との区別を名前で明示する。

### `mypage/profile/`, `mypage/account/` を昇格
`/mypage/profile` → `/profile`、`/mypage/account` → `/account` へ。  
マイページ配下かどうかはナビゲーションで表現し、URL 階層に依存しない。
