# netherid UI設計書

## 1. デザインコンセプト
「一元管理の利便性」と「個人情報を預ける安心感・信頼感」を両立させた、プロフェッショナルで洗練された世界観を提供します。
MUUUUU.ORG等に掲載されるような、クリーンでモダン、かつセキュリティの高さを感じさせるSaaS向けのデザインシステムを採用します。

## 2. カラースキーム（カラーパレット）
ご指摘の通り、多色使いは視認性を下げデザインの難易度を上げるため、**「ブランドカラー（安心感のグリーン）＋グレースケール」**という非常にシンプルで洗練された構成に絞り込みます。グリーンは心理学的に「安全・安心・平和」を感じさせるため、個人情報を預けるID管理システムに非常に適しています。

- **Brand & Primary (ブランド＆主要アクション):** `#059669` (Emerald 600 - 深みと鮮やかさを両立したグリーン)
  - サービスロゴ、主要ボタン（保存・認証など）、アクティブなリンク、入力フォーカス時のリングなどに使用し、ユーザーを安全に導きます。
- **Grayscale (グレースケール - 画面の骨格とテキスト):**
  - **Text Main (見出し・主要テキスト):** `#111827` (Gray 900 - 視認性の高い濃いグレー)
  - **Text Muted (補助テキスト・グレーアウト):** `#6B7280` (Gray 500)
  - **Background (画面背景):** `#F9FAFB` (Gray 50 - 圧迫感のない極めて薄いグレー)
  - **Surface (カード背景):** `#FFFFFF` (White)
  - **Border & Gray-out (枠線・グレーアウト背景):** `#E5E7EB` (Gray 200)
- **Accent (注意・警告用):** `#DC2626` (Red 600)
  - 退会ボタンやエラーメッセージなど、ユーザーに注意を促す必要のある箇所に限定して使用します。

## 3. 基本レイアウト構成と世界観の反映

### 3.1. パブリック画面（LP・登録・ログイン等）
- **ヘッダー:** 白背景（`#FFFFFF`）にブランドカラーのロゴ。シンプルで清潔感のある入り口とします。
- **メインエリア:** `#F9FAFB` (背景色) の上に、`#FFFFFF` のログイン/登録用カードを中央配置。
- **フッター:** クリーンな印象を保つため、白または薄いグレー背景で統一します。

### 3.2. マイページ画面（ログイン後）
一元管理の「見通しの良さ」を強調する2カラムレイアウト。

- **トップヘッダー:** 白背景（`#FFFFFF`）の下部に極薄いボーダー（`#E5E7EB`）。スッキリとしたヘッダーで圧迫感をなくす。
- **左サイドバー（ナビゲーション）:**
  - 画面左側に固定。背景色は `#F9FAFB`。
  - アクティブなメニューは、左側に Brand Color (`#059669`) の細いラインを引き、テキストを濃くする等の上品なハイライト。
  - **※グレーアウト項目**は `#6B7280` で表示し、小さな「Lock」アイコン等を添えて、将来的な拡張性を期待させつつセキュアに保護されている印象を与えます。
- **メインコンテンツ（右側）:**
  - 画面上部にページタイトル（`#111827`）と簡単な説明文。
  - 機能ごとの**カード（Surface: `#FFFFFF`）**。カード間には十分な余白を持たせ、整理整頓された印象を与えます。

## 4. 主要コンポーネントデザイン

### 4.1. ボタン (Buttons)
角丸は `6px` 〜 `8px` 程度で統一し、モダンで押しやすい形状に。
- **Primaryボタン:** 背景 `#059669`、文字 `#FFFFFF`。ホバー時は `#047857`（少し濃いグリーン）へ滑らかに遷移。
- **Outlineボタン:** 背景透明、枠線 `#E5E7EB`、文字 `#111827`。
- **Disabled（非活性）ボタン:** 背景 `#E5E7EB`、文字 `#6B7280`。クリック不可。

### 4.2. 入力フォーム (Inputs)
- 背景 `#FFFFFF`、枠線 `#E5E7EB`。
- フォーカス（入力中）時は、枠線を `#059669` に変え、周囲に薄いグリーンのリング（`box-shadow: 0 0 0 3px rgba(5, 150, 105, 0.2)`）を展開。入力中の安心感・確実性を提供する。

### 4.3. パネル・カード (Cards)
- `border-radius: 12px`、ボーダーは `#E5E7EB` の1px実線。
- シャドウは多用せず、フラットでクリーンな「データ管理ダッシュボード」のテイストを重視する。

### 4.4. グレーアウト（未実装機能）の表現
単に「使えない」というネガティブな印象を避けるため、以下のようにデザインする。
- パネル全体を `#F9FAFB` (Gray 50) で塗りつぶし、不透明度 `0.7`。
- 項目の中央や右上に、セキュアな印象を与える「Lockアイコン」と「準備中 (Coming Soon)」の小さなバッジ（背景 `#E5E7EB`、文字 `#6B7280`、角丸フル）を配置。
- 「今後ここから管理できるようになる」という一元管理の全体像をユーザーに意識させる役割を持たせる。
