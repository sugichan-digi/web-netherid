# デザインシステム設計書

本ドキュメントは、NetherIDプロジェクトにおけるフロントエンド共通デザイン（デザインシステム）の実装方針と、共通CSS（`design-system/style.css`）に定義すべき内容をまとめたものです。

## 1. デザインシステムの目的と前提条件

本プロジェクトではモダンなフレームワーク（React/Vue）やCSSフレームワーク（Tailwind CSS/Bootstrap）、メタ言語（SCSS/SASS）を**一切使用しません**。
純粋な **HTML と Vanilla CSS** だけで、高い保守性と一貫性を持つデザインシステムを構築します。

### 前提条件・実装ルール
1. **CSS変数の徹底活用**: 色、余白、フォントサイズなどの「デザイントークン」は、すべてルート（`:root`）のCSS変数（カスタムプロパティ）として定義し、ハードコードを避けます。
2. **ユーティリティとコンポーネントの分離**: 
   - 基本的なUI（ボタン、フォーム、カード）は**コンポーネントクラス**（例: `.btn`, `.card`）として定義します。
   - 余白調整などの微調整には、最低限の**ユーティリティクラス**（例: `.mt-4`, `.text-center`）を用意します。
3. **カスケードの最小化**: BEMライクな命名規則や、セレクタのネストを浅く保つことで、意図しないスタイルの上書き（詳細度地獄）を防ぎます。
4. **アイコンのSVG化**: 絵文字は一切使用せず、アイコンはすべてSVG形式で実装します。

## 2. 定義すべきデザイントークン（CSS変数）

`design-system/style.css` の `:root` セクションに定義する変数群です。

### 2.1. カラーパレット (Colors)
UI設計書に基づいた「グリーン＋グレースケール＋アクセント」の構成です。

```css
:root {
  /* Brand / Primary (安心感のグリーン) */
  --color-primary-main: #059669;    /* Emerald 600: メインボタン等 */
  --color-primary-hover: #047857;   /* Emerald 700: ホバー時 */
  --color-primary-light: #d1fae5;   /* Emerald 100: 薄い背景等 */

  /* Grayscale (骨格とテキスト) */
  --color-gray-50:  #f9fafb; /* 画面の背景色 */
  --color-gray-200: #e5e7eb; /* ボーダー、非活性背景 */
  --color-gray-500: #6b7280; /* 補助テキスト、アイコン */
  --color-gray-900: #111827; /* 見出し、主要テキスト */
  --color-white:    #ffffff; /* カード背景 */

  /* Accent (エラー・警告) */
  --color-error: #dc2626; /* Red 600 */
}
```

### 2.2. タイポグラフィ (Typography)
視認性を重視したシステムフォントスタックと、階層化されたサイズ。

```css
:root {
  --font-family-base: 'Inter', 'Helvetica Neue', Arial, 'Hiragino Kaku Gothic ProN', 'Hiragino Sans', Meiryo, sans-serif;
  
  --text-xs: 0.75rem;   /* 12px */
  --text-sm: 0.875rem;  /* 14px */
  --text-base: 1rem;    /* 16px */
  --text-lg: 1.125rem;  /* 18px */
  --text-xl: 1.25rem;   /* 20px */
  --text-2xl: 1.5rem;   /* 24px */
}
```

### 2.3. スペーシング (Spacing)
8pxベースのグリッドシステムを採用し、一貫した余白を作ります。

```css
:root {
  --space-1: 0.25rem; /* 4px */
  --space-2: 0.5rem;  /* 8px */
  --space-3: 0.75rem; /* 12px */
  --space-4: 1rem;    /* 16px */
  --space-6: 1.5rem;  /* 24px */
  --space-8: 2rem;    /* 32px */
}
```

### 2.4. その他 (Shadow, Radius)
```css
:root {
  --radius-sm: 4px;
  --radius-md: 6px;
  --radius-lg: 12px;
  --radius-full: 9999px;

  --shadow-sm: 0 1px 2px 0 rgba(0, 0, 0, 0.05);
  --shadow-md: 0 4px 6px -1px rgba(0, 0, 0, 0.1), 0 2px 4px -1px rgba(0, 0, 0, 0.06);
}
```

## 3. 共通コンポーネントクラス群

各画面で使い回す基本UIパーツのクラス名定義です。

### 3.1. ボタン (`.btn`)
* `.btn`: 基本構造（パディング、角丸、トランジション、フォントサイズ等）
* `.btn-primary`: 背景グリーン、文字白
* `.btn-outline`: 背景透過、枠線グレー、ホバーで薄いグレー
* `.btn-disabled`: 背景・文字色グレー、カーソル操作不可

### 3.2. フォーム (`.form-control`, `.form-group`)
* `.form-group`: ラベルと入力欄、エラーメッセージをまとめるラッパー（縦並び、余白設定）。
* `.form-label`: 入力欄のラベル（太字、ダークグレー）。
* `.form-control`: `<input>`, `<select>` 等の基本スタイル。
  * `&:focus`: 枠線を `--color-primary-main` に変更し、薄い緑の `box-shadow` を付与。

### 3.3. レイアウト・カード (`.container`, `.card`)
* `.container`: 最大幅の制限（例: `max-width: 1024px;`）と中央揃え。
* `.card`: 白背景、角丸（`--radius-lg`）、薄い枠線（`--color-gray-200`）、必要に応じたシャドウ。
  * グレーアウト時はクラス `.card-disabled` などを付与し、不透明度とオーバーレイを設定。

### 3.4. バッジ (`.badge`)
* `.badge`: 小さな角丸要素。ステータス表示（準備中、認証済みなど）に使用。

## 4. アイコンの管理と実装方法

**絵文字（Emoji）は使用禁止**です。アイコンは必ずSVGを使用します。

1. **インラインSVGの利用**:
   軽量なアイコン（矢印、ロックアイコン、チェックマークなど）は、HTML内に直接 `<svg>` タグを埋め込みます。これによりCSSの `currentColor` を用いて色を動的に変更できます。
   
2. **アセットディレクトリでの管理**:
   共通で使うSVGファイルは `frontend/assets/icons/` 等のディレクトリに配置し、`<img>` タグ等で呼び出すことも可能です（ただし色の動的変更が必要な場合はインラインSVGを推奨します）。

---

## 5. 次のステップ（実装）

本設計書に基づき、`frontend/design-system/style.css` を作成し、上記すべての変数（CSSカスタムプロパティ）と共通コンポーネントクラスを定義します。その後、各画面のHTMLを作成する際にこのCSSを読み込んでUIを構築します。
